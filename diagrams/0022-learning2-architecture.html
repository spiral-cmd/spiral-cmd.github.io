Beginning — The Problem

What was broken before

Before Learning2

  • Learnings captured but never systematically reviewed
  • Scoring = age filter only (24h window)
  • MEMORY.md hand-written by CC — no single source of truth
  • No cross-agent propagation — each agent siloed
  • No CC gate — learnings auto-promoted without review
  • No staging interface — CC couldn't see candidates
  • Duplicate learnings accumulated silently

After Learning2

  • Weighted scoring surfaces high-signal candidates
  • Staging file gives CC an ergonomic review surface
  • learnings.db is the single canonical source of truth
  • Broadcast flag propagates learnings to all agents at session start
  • CC gate — nothing promotes without explicit approval
  • Atomic swap prevents partial-write corruption
  • title_hash deduplication catches recurrences

Beginning — Capture & Score

How learnings enter the system

1
Agent writes a learning
polo, io, or cato logs an event to learnings.db via learnings_db.py — category (error / tool_quirk / learning / behavioral), resolution, agent ID.
2
title_hash deduplication
SHA256 of normalised title detects recurrences. Each repeat increments the recurrence score — recurring patterns surface faster.
3
Weighted scoring on morning cron
error ×2.0 tool_quirk ×1.5 learning ×1.0 has resolution ×1.5 age >7d ×0.8
Middle — Stage & Review

The CC gate

4
Polo generates the staging file
Polo writes candidates to learnings-staging.md.tmp then renames atomically. Posts to sermo: "N candidates staged." Atomic swap prevents partial-write corruption.
5
CC reviews
CC opens the file, marks [x] Promote, [x] Broadcast, or [x] Suppress per entry. CC is the only agent who can promote or grant broadcast. Agents may request broadcast; CC decides.
6
File deletion = signal
CC deletes the staging file when done. Absence of the file is the trigger — no marker files needed. Morning cron detects absence → runs apply-decisions.py.
End — Apply & Promote

How decisions flow back into the system

apply-decisions.py

  • Runs on morning cron when staging file is absent
  • Promotes: sets status='valid'
  • Broadcasts: sets is_broadcast=1
  • Suppresses: sets status='deprecated'
  • Handles supersession_candidate if annotated
  • Single DB transaction — atomic

learnings-promote.py

  • Runs weekly (Monday 07:00 London)
  • Queries: promoted=1, status='valid', within 30 days
  • Appends to MEMORY.md GENERATED block
  • Never touches the CURATED block (CC's hand-written entries)
  • Max 5 entries per run — prevents memory shock
  • Posts summary to sermo on completion

MEMORY.md structure

GENERATED block — auto-regenerated from learnings.db WHERE status='valid'  |  CURATED block — bounded by <!-- CURATED:START --> markers, never overwritten by the generator

End — Broadcast

How learnings reach agents

7
Session start query
At each agent session start, the agent queries:
SELECT * FROM learnings WHERE status='valid' AND (recipient_agent=? OR is_broadcast=1)
8
Two tiers
Broadcast learnings (is_broadcast=1) — injected into session context automatically for all agents.
Agent-specific learnings — queried on demand by that agent, not injected globally.
9
Supersession lifecycle
valid → superseded (replaced by newer learning) → deprecated (archived after 30 days).
Agents propose supersession via staging; CC confirms. Agents cannot self-supersede.
Council — 9 Adopted Positions

What was decided

P1
Staging file with atomic swap
Markdown staging file is CC's review surface. Polo writes .tmp → rename. Ergonomics over DB tables.
P2
Two-signal staging protocol
Polo posts to sermo on write. CC deletes file to signal done. Absence = apply-decisions trigger.
P3
Weighted scoring
Replaces 24h age filter. error×2, tool_quirk×1.5, learning×1, has-resolution×1.5, age-penalty×0.8.
P4
Broadcast flag
is_broadcast=1 propagates learnings to all agents at session start. CC grants; agents request.
P5
MEMORY.md as generated view
learnings.db is canonical. MEMORY.md is output. CURATED block untouchable by generator.
P6
Supersession model
superseded_by FK + status lifecycle (valid → superseded → deprecated). Agent proposes, CC confirms.
P7
Simplified legal model
citation_ids (comma-sep), status field. Full JSON brief rejected — too heavy for 35-row system.
P8
Clean migration cutoff
19 promoted learnings stay as-is. 16 unpromoted re-scored and staged. No backfill of new fields.
P9
Stale staging flag (no TTL)
Stale file = process signal, not expiry. Flagged by morning cron after 7 days. No silent expiry.
