Control Gates
CIV Gate P0

Enforces CC approval between phases. IO cannot send Polo the next work package until CC sends an explicit ACK. Two independent timeout chains (see lifecycle §9 above).

"vai" Gate P1

Irreversible actions (config changes, upgrades, DB ops) require Martin's direct approval via Telegram — the word vai. Not relayed through agents. Not via files. HITL checkpoint, no bypass.

Comms & Visibility
hard rule Every message CC sends to Polo or IO must end with "Reply via mailbox and sermo" — no exceptions.
to-cc.md
return path
Polo and Cato write replies here. CC reads at session start (hook) or via /csn mid-session. Archived to to-cc-log.md before clearing.

Auto-ACK watcher: launchd WatchPaths on mailbox.sqlite — fires to-cc-watcher.sh, posts [CC] Auto-ACK to sermo within ~90 s. High-priority messages also posted to topic 1.
a2alog
shared context
Every substantive sessions_send logged to message_log. Agents in Topic 10 read last 8 entries on first turn. Not just logging — it's the shared context layer that makes multi-agent coordination coherent across session boundaries.
sermo
visibility only
Telegram topic 111 in ocaisa group. Martin's visibility layer — not an action channel. Polo does not act on [CC] sermo posts by design. CC posts after every direct agent message so Martin can see what was asked. Polo posts [Polo] prefix for same reason.
mailbox.sqlite
persistent queue
Durable message store. ACK watcher monitors this file via WatchPaths. Survives session restarts. Used alongside to-cc.md — mailbox for delivery guarantees, to-cc for human-readable queue CC reads at startup.
