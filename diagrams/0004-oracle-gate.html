Plugin Details

Hook before_tool_call

  • Fires synchronously before any tool executes
  • Priority 100 — runs ahead of lower-priority hooks
  • Returns { block: true, blockReason: "..." } to stop execution
  • Returns nothing to allow

Gate logic Evidence contract

  • Gated tools: sessions_send, sessions_spawn
  • Evidence path: /tmp/evidence-{agentId}-{taskId}
  • No taskId in session → gate is skipped (cron/heartbeat safe)
  • Empty file counts as missing — agent must write real content

Discovery Plugin setup

~/.openclaw/extensions/oracle-gate/ index.js ← hook logic openclaw.plugin.json package.json ~/.openclaw/node_modules/oracle-gate → symlink to extensions/oracle-gate

Integration Agent side

  • Before reporting done, agent writes evidence to /tmp/evidence-{agentId}-{taskId}
  • Content: tool call result, file write confirmation, or message ID
  • Soft gate (AGENTS.md rule) + hard gate (this plugin) work in tandem
  • Connects to io's mandatory evidence requirement added 2026-05-11

Replaced oracle-gate-test

  • Previous test plugin blocked all exec calls unconditionally
  • Removed from plugins.allow on 2026-05-11
  • oracle-gate targets only delegation tools, not execution tools
  • Exec calls from Polo/Cato are unaffected by this gate

Context Why it exists

  • FM-5 failure pattern: agent claims delegation but makes no tool call
  • io wrote "Codo has GO" with zero tool calls — Codo was never briefed
  • Oracle Gate makes undocumented delegation physically impossible
  • Part of Tier 1 agent-reliability roadmap (2026-05-11)
