Key Findings
⚠️
Per-agent compaction is schema-locked
OpenClaw intentionally strips compaction from agents.list[] entries. Defaults-only. GitHub issue #57174 tracks the feature request. Workaround: tune defaults for tightest agent; large-context agents absorb the conservatism.
🔍
forceFlushTranscriptBytes was dead for Cato
600kb ≈ 150k tokens of text — above Cato's 128k window. The backstop would never fire. softThresholdTokens was the only active guard, and it was set too late (15k → 88% utilisation).
📋
Research source: Perplexity doc 02
Per-agent override confirmed intentionally unsupported by official docs + GitHub issues #13736, #52732, #57174. contextLimits.postCompactionMaxChars is the one supported per-agent lever.
📐
Gemma 4 31B → V4 Flash migration
Polo migrated 2026-04-30 from Gemma 131k to DeepSeek V4 Flash 1M due to context saturation. The compaction thresholds set for 131k became wildly inappropriate for 1M without adjustment.
