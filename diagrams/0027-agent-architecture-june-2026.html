Workplan Convention (Position 4)
🔭 Scout Mode — Design & Planning
· Scope is provisional
· Steps change as you learn
· Map drawn as you go
· No strict state tracking
· Brief is the deliverable
🔑
Switching signal: CC says vai on the brief.
Polo writes Current phase: implementation to state header.
⚙️ State Machine — Delivery
· Scope is fixed
· Deviations must be flagged
· State header mandatory
· Phase transitions logged
· Blocked on → CC check ≤2h
State header (every workplan at project/[name]/workplan.md):

## State
Current phase: [design | planning | implementation | review | done]
Blocked on: [description or "none"]
Cross-layer deps: [agents waiting, or "none"]
Last updated: [ISO date]
Minimum Task Completion Report (Position 6)
Required fields (all tasks)
Completed: bullet list of changes
Wall-clock time: active / elapsed if different
Not done: deferred/deprioritised scope
Not verified: claims without direct evidence
Blockers: anything preventing further progress
What this prevents
"Done in 7 minutes" + 3 items in Not done = visible contradiction

Applies to: Polo→CC, Polo→io, io→CC, CC→Martin briefings
