Codex Ingest Pipeline — Full Flow

From browser save to searchable document, including Martin's manual tags and the orchestrator pipeline

📥 Ingestion (Hazel + tag-learn)
1
Source

PDF Saved from Browser

Martin saves a PDF from the browser. If he adds a Finder tag (e.g. "ai", "philosophy", "tosort") at save time, that tag travels with the file through the pipeline.

~/Downloads/  →  macOS Finder tag applied at save time
2
Hazel

Hazel Rule → pdf-in/

Hazel watches Downloads. On new PDF, it prepends the date (yyyymmdd) to the filename and moves it to pdf-in/. The macOS Finder tag survives the move (Hazel preserves xattrs).

Downloads/paper.pdf → pdf-in/20260621 paper.pdf  (tag: "ai")
3
Watcher

tag_watcher.sh Fires

launchd WatchPaths triggers tag_watcher.sh. It first checks process_tagged.py — if the file already has a valid Finder tag (added by Martin at save time), it's processed silently with no UI.

process_tagged.py → exit 0 (tagged) or exit 1 (no tag → continue)

process_tagged.py routes it directly → ingest

tag_learn.py ≥ 0.85 → auto-tag → ingest

4a
Review

Martin Reviews in Tag UI

For low-confidence suggestions, tag_server.py launches a browser UI at http://127.0.0.1:5555. Martin picks a tag, sets provenance flags, and confirms. tag_worker.py handles the rest.

Tag grid → confirm → file moves to codex/<tag>/ → ingest
4b
Ingest

Ingest to codex.db

Whether auto-tagged, Martin-tagged, or UI-confirmed, the file is moved to /Volumes/4tb-a/codex/<tag>/ via ditto (preserving xattrs). codex_log is written, vector store updated, and codex_docs gets a row with cache_state='indexed'.

cache_state = 'indexed'  ·  text extracted  ·  FTS populated  ·  0 chunks
⚙️ Orchestrator Pipeline (codex-orchestrator.py)
1
Detect

Detect Indexed Docs with 0 Chunks

Scans codex_docs for documents where cache_state='indexed' and no chunks exist. These are fully ingested but not yet chunked or vector-embedded — the orchestrator's job.

SELECT ... WHERE cache_state='indexed' AND NOT EXISTS (SELECT 1 FROM chunks ...)
2
Chunk

Chunking

Reads full text from codex_fts (or text_cache_path fallback). Splits into overlapping 1000-character chunks with 200-character overlap. Inserts into chunks table. Sets cache_state → 'chunked'.

chunk_size=1000 · overlap=200 · INSERT INTO chunks (document_id, chunk_index, start, end, text)
3
Embed

Vector Embedding

For each chunk, generates a 384-dim vector embedding using all-MiniLM-L6-v2 via embedding.embed_text(). Inserts into codex_vec virtual table. Sets cache_state → 'embedded'.

embedding.embed_text(chunk_text) → INSERT INTO codex_vec (chunk_id, embedding)
4
Enrich

Enrichment

Two parallel enrichment passes:

• Entity extraction (spaCy en_core_web_lg) — extracts PERSON, ORG, GPE, LOC entities → entities table

• Concept fingerprinting (KeyBERT MMR) — extracts top-10 keyphrases → concept_fingerprints table

Sets cache_state → 'complete'.

spaCy nlp(text[:10000]) · KeyBERT(text[:10000], top_n=10, diversity=0.5)
5
Done

Pipeline Complete

Reports summary to Sermo (topic 111): documents attempted, chunked, embedded, enriched, and failed. Releases file lock. Ready for next batch.

[codex-orchestrator] Auto run: 180/180 complete (0 failed)

🚀 Auto Mode

--auto [--batch-size 50]

Detects newly indexed docs and runs the full pipeline.

🔄 Resume Mode

--resume [--batch-size 50]

Finds stuck docs (chunked/embedded/failed) and resumes from correct stage.

📊 Status

--status

Shows pipeline state distribution, lock status, and targets.

🏷 Martin's Manual Tags — When Martin saves a PDF from the browser and adds a Finder tag (e.g. "ai", "philosophy", "tosort"), that tag is preserved through Hazel's date-prepend move. process_tagged.py detects it immediately and routes the file directly to the correct codex folder — no ML suggestion, no UI, no delay. The orchestrator then picks it up for chunking/embedding/enrichment.
🔒 File Lock — codex-orchestrator.lock prevents concurrent runs. 5-minute timeout with stale lock detection. Released automatically on completion.
🔄 Resume Logic — On --resume, documents in 'failed' state are checked for existing chunks. If chunks exist → resume from embedding. If no chunks → reset to 'indexed' and let auto mode handle it. Idempotent — safe to rerun.
