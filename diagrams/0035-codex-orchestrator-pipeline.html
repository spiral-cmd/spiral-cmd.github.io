Codex Ingest Pipeline Orchestrator

End-to-end pipeline: auto-tagged file → chunking → embedding → enrichment → complete

🚀 Auto Mode

--auto [--batch-size 50]

Detects newly auto-tagged files (cache_state='indexed', 0 chunks) and runs the full pipeline.

🔄 Resume Mode

--resume [--batch-size 50]

Finds documents stuck in 'chunked', 'embedded', or 'failed' states and resumes from the correct stage.

📊 Status

--status

Shows pipeline state distribution, total docs/chunks, lock status, and pipeline targets.

Trigger

Detect New Auto-Tagged Files

Scans codex_docs for documents with cache_state='indexed' that have zero chunks. These are newly auto-tagged PDFs ready for pipeline processing.

SELECT ... WHERE cache_state='indexed' AND NOT EXISTS (SELECT 1 FROM chunks ...)
acquire file lock →
2
Chunk

Chunking

Reads full text from codex_fts (or text_cache_path fallback). Splits into overlapping 1000-character chunks with 200-character overlap. Inserts into chunks table. Sets cache_state → 'chunked'.

chunk_size=1000 · overlap=200 · INSERT INTO chunks (document_id, chunk_index, start, end, text)
3
Embed

Vector Embedding

For each chunk, generates a 384-dim vector embedding using all-MiniLM-L6-v2 via the existing embedding.embed_text() module. Inserts into codex_vec virtual table. Sets cache_state → 'embedded'.

embedding.embed_text(chunk_text) → INSERT INTO codex_vec (chunk_id, embedding)
4
Enrich

Enrichment

Two parallel enrichment passes:

• Entity extraction (spaCy en_core_web_lg) — extracts PERSON, ORG, GPE, LOC entities → entities table

• Concept fingerprinting (KeyBERT MMR) — extracts top-10 keyphrases → concept_fingerprints table

Sets cache_state → 'complete'.

spaCy nlp(text[:10000]) · KeyBERT(text[:10000], top_n=10, diversity=0.5)
5
Done

Pipeline Complete

Reports summary to Sermo (topic 111): documents attempted, chunked, embedded, enriched, and failed. Releases file lock. Ready for next batch.

[codex-orchestrator] Auto run: 180/180 complete (0 failed)
🔒 File Lock — codex-orchestrator.lock prevents concurrent runs. 5-minute timeout with stale lock detection (checks if holding PID is still alive). Released automatically on completion.
🔄 Resume Logic — On --resume, documents in 'failed' state are checked for existing chunks. If chunks exist → resume from embedding. If no chunks → reset to 'indexed' and let auto mode handle it. Idempotent — safe to rerun.
