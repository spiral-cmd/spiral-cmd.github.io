Polo Productivity Protocol

CC → io — active for duration of FT scrape task
Hard Rules
Rule 1
One clear task at a time
Brief Polo with full spec and explicit success criteria. No ambiguity about what "done" looks like.
Rule 2
Timeboxes enforced
Each step has a time estimate. If Polo exceeds it, stop and report — do not keep trying.
Rule 3
2-attempt blocker threshold
Any step failing twice → Polo stops immediately and reports to io-inbox + to-cc + sermo. No silent retries.
Rule 4
Pre-check is blocking
If Firefox is running, Polo posts to sermo and waits for Martin to confirm it's closed. Does not proceed.
Rule 5
No scope creep
Polo builds the script and runs once. Structural issues → document and escalate. Do not redesign mid-task.
Rules 6 & 7
Strict report format
Completion and failure each have a defined format. See below.
Report Formats

✓ On completion

  • Selector used + article count
  • Date extraction method
  • Total articles after date filter
  • Path to ft_results.json
  • Any warnings / partial failures

✗ On failure

  • Step that failed
  • Error message verbatim
  • Number of attempts made
  • What was tried
  • Recommended next step (do not self-assign)
io: Monitor sermo during execution. Unblock within authority or escalate to CC. Do not re-brief Polo without CC approval.
